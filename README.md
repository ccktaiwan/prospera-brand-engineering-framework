====================================================================

1. PURPOSE

This specification defines the canonical Prospera Brand Engineering 
Framework. It translates consulting expertise into a deterministic, 
phase-gated system designed to transition SMEs from founder-dependent 
growth to scalable, system-driven operations.

====================================================================

2. SYSTEM POSITIONING (NORMATIVE)

- R-01: This repository operates as a Productized Execution Framework 
  subordinate to Prospera-Governance-Core.
- R-02: Every brand engineering mission MUST follow the 6-phase 
  lifecycle without bypass.
- R-03: The Framework SHALL NOT define governance authority but 
  MUST implement governance-locked decisions.

====================================================================

3. FRAMEWORK INVARIANTS (HARD CONSTRAINTS)

- I-01: EVIDENCE_SUPREMACY: Brand decisions MUST be derived from 
  observable signals (Phase 1-3) rather than intuition or preference.
- I-02: PHASE_ISOLATION: No brand signal amplification (Phase 4) is 
  permitted until the Brand Reality Baseline (Phase 1) is verified.
- I-03: FOUNDER_DECOUPLING: The objective of all framework 
  activities is the externalization of brand knowledge from 
  individual memory to system structure.

====================================================================

4. FAILURE MODES & DEGRADATION

- F-01: Aspiration Over Reality -> If brand signals outpace delivery 
  evidence, the system MUST trigger a Phase 3 Reversion.
- F-02: Governance Fatigue -> If oversight cost increases during 
  scaling, the transition to Phase 6 SHALL be blocked.

====================================================================

5. TRAVERSAL RULE

All implementation and AI-assisted branding tasks MUST begin 
discovery from the `SYSTEM_INDEX.md` within this framework.
